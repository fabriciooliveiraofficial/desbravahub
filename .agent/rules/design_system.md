# Rule: Design System Enforcement

All user interface changes, new pages, and component modifications MUST strictly adhere to the [Master Design System & Style Guide](file:///C:/Users/User/.gemini/antigravity/brain/ef0b8ee2-d679-4bcb-a91c-268dbad7fd60/master_design_guide.md).

## Key Directives

1.  **Variables Only:** NEVER use hardcoded colors, paddings, or border-radii. Always use the specified CSS variables (e.g., `var(--color-accent)`, `var(--radius-xl)`).
2.  **Iconography:** Use **Iconify** for all icons. Prefer thin-line styles (Lucide or Phosphor-Thin).
3.  **Consistency:** Ensure that modals, forms, and alerts match the established visual language (Glassmorphism, specific shadows, and Inter/Outfit typography).
4.  **Aesthetics:** Every UI change must feel premium and professional. Avoid generic AI-generated aesthetics.

Consult the guide before starting any frontend implementation.
